### **Interacción 5: Consulta sobre publicación en GitHub**

*   **Usuario:**
    > "ME GUSTO, AHORA COMO LO SUBO A GITHUB?"

*   **Inteligencia Artificial (Antigravity):**
    > *[Proceso interno: Creación preventiva de documentación complementaria (`README.md` detallando la propuesta y `.gitignore` para evitar archivos temporales del sistema) para dejar el proyecto listo para su portafolio].*
    >
    > "Para subir tu proyecto a GitHub y tenerlo guardado en la nube (e incluso **hacerlo público para que cualquiera pueda jugar con un enlace de internet gratis** usando GitHub Pages), puedes seguir cualquiera de estos métodos.
    > 
    > Te recomiendo especialmente el **Método A** (usando la terminal si ya tienes Git instalado) o el **Método B** (usando la página web de GitHub, que es el más rápido sin instalar nada).
    > 
    > ---
    > 
    > ### Paso Previo: Crear el Repositorio en GitHub
    > 1. Inicia sesión en [GitHub](https://github.com/) (crea una cuenta si aún no tienes una).
    > 2. En la esquina superior derecha, haz clic en el botón **`+`** y selecciona **New repository** (Nuevo repositorio).
    > 3. Escribe un nombre para tu proyecto (por ejemplo: `domino-numerico-infantil`).
    > 4. (Opcional) Agrega una breve descripción.
    > 5. Elige si quieres que sea **Público** (cualquiera puede verlo) o **Privado** (solo tú). *Nota: Para usar la web gratis con un link público, debe ser público.*
    > 6. **MUY IMPORTANTE**: Deja desmarcadas las opciones de *Add a README file*, *Add .gitignore* y *Choose a license*, ya que nosotros ya creamos estos archivos en tu computadora.
    > 7. Haz clic en **Create repository** (Crear repositorio).
    > 
    > ---
    > 
    > ### Método A: Subirlo desde la Terminal (Recomendado si usas Git)
    > Abre la terminal o PowerShell en la carpeta del proyecto (`c:\Users\ramuj\OneDrive\Documentos\PRUEBA 2`) y ejecuta los siguientes comandos uno a uno:
    > 
    > ```bash
    > # 1. Inicializa el repositorio local de Git
    > git init
    > 
    > # 2. Agrega todos los archivos al área de preparación
    > git add .
    > 
    > # 3. Haz tu primer guardado (commit)
    > git commit -m "Primer commit: Dominó Numérico Infantil"
    > 
    > # 4. Cambia la rama principal a "main"
    > git branch -M main
    > 
    > # 5. Vincula tu repositorio local con el de GitHub
    > # (Copia la línea exacta que te aparece en la página de GitHub al crear el repositorio)
    > git remote add origin https://github.com/TU_USUARIO/TU_REPOSITORIO.git
    > 
    > # 6. Sube los archivos a GitHub
    > git push -u origin main
    > ```
    > 
    > ---
    > 
    > ### Método B: Subirlo Directamente desde la Web (Sin instalar nada)
    > Si prefieres no usar comandos ni terminal, GitHub te permite arrastrar los archivos:
    > 1. En la página del repositorio vacío que acabas de crear, busca un enlace que dice **"uploading an existing file"** (subir un archivo existente) en la sección central.
    > 2. Arrastra los archivos de tu computadora (`index.html`, `style.css`, `app.js`, `README.md` y `.gitignore`) a la zona de carga de la página web.
    > 3. Espera a que se carguen.
    > 4. En la parte inferior, escribe un mensaje de confirmación (ej: `Subir dominó numérico`) y haz clic en **Commit changes** (Confirmar cambios).
    > 
    > ---
    > 
    > ### 🎁 ¡Extra! Cómo hacer que el juego funcione online gratis (GitHub Pages)
    > Una vez que tus archivos estén en GitHub, puedes hacer que el juego funcione en una página web real con un enlace para que los niños o maestros jueguen desde tablets u otros dispositivos:
    > 
    > 1. Dentro de tu repositorio en GitHub, ve a la pestaña **Settings** (Configuración) en la parte superior.
    > 2. En la barra lateral izquierda, haz clic en **Pages** (Páginas).
    > 3. En la sección **Build and deployment**, bajo **Branch**, cambia el selector que dice *None* por **`main`** y deja la carpeta en **`/ (root)`**.
    > 4. Haz clic en **Save** (Guardar).
    > 5. Espera aproximadamente 1 minuto. Al recargar la página de *Pages*, aparecerá un banner arriba que dice: **"Your site is live at..."** seguido de un enlace (ej: `https://tu-usuario.github.io/domino-numerico-infantil/`).
    > 6. ¡Comparte ese link para que cualquiera juegue desde su celular, tableta o computadora!"
